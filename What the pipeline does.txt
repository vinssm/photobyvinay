How to deploy to Azure
Architecture
The React app only ever talks to one domain — the main App Service handles routing to the microservices.

Step 1 — Create 3 Azure App Services
In the Azure Portal:

App	Runtime	Suggested name
Main (GraphQL + React)	Node.js 20	photobyvinay
Bio Service	Node.js 20	photobyvinay-bio
Project Service	Python 3.11	photobyvinay-projects
For each one: App Service → Get publish profile → download the .PublishSettings file.

Step 2 — Set App Settings in Azure Portal
Main App Service → Configuration → Application Settings:

Key	Value
NODE_ENV	production
ATLAS_URI	your MongoDB Atlas URI
JWT_SECRET	a strong secret string
BIO_SERVICE_URL	https://photobyvinay-bio.azurewebsites.net
PROJECTS_SERVICE_URL	https://photobyvinay-projects.azurewebsites.net
Startup command	node server/server.js
Bio Service → Configuration:

Key	Value
NODE_ENV	production
Startup command	node server.js
Project Service → Configuration:

Key	Value
Startup command	uvicorn main:app --host 0.0.0.0 --port 8000
Step 3 — Add GitHub Secrets & Variables
Go to your GitHub repo → Settings → Secrets and variables → Actions:

Secrets (paste content of each .PublishSettings file):

Secret name	Value
AZURE_MAIN_PUBLISH_PROFILE	contents of main app publish profile
AZURE_BIO_PUBLISH_PROFILE	contents of bio service publish profile
AZURE_PROJECTS_PUBLISH_PROFILE	contents of project service publish profile
Variables (Repository variables tab):

Variable name	Value
AZURE_MAIN_APP_NAME	photobyvinay
AZURE_BIO_APP_NAME	photobyvinay-bio
AZURE_PROJECTS_APP_NAME	photobyvinay-projects

Step 4 — Push to main

git add .
git commit -m "deploy: add Azure CI/CD for all 3 services"
git push origin main


GitHub Actions will run 3 parallel jobs and deploy each service independently. Every future push to main redeploys all three automatically.
